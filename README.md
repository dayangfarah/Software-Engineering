## 📖 Course Synopsis
This course is designed to give students an introduction to an engineering approach in the development of high-quality software systems. It will discuss the important software engineering concepts in the various types of the common software process models. The students will also learn the concepts and techniques used in each software development phase including requirements engineering, software design and software testing. This course will also expose the students to utilizing object-oriented method (e.g. UML) and tools in analyzing and designing the software. At the end of this course, students are expected to be able to appreciate most of the common software engineering concepts and techniques as well as producing various software artifacts and deliverables.

## 💭 Reflection

### What I Learned
Taking SECP 3204 completely reframed my perspective on building software systems, shifting my focus from unstructured scripting to structured engineering principles. I learned to systematically navigate different lifecycle pathways, drawing clear contrasts between plan-driven approaches and agile methodologies. Delving into the concrete phases of engineering taught me how to translate abstract user needs into functional requirements, build comprehensive architectural models, and establish structured testing plans. The exposure to object-oriented methods and Unified Modeling Language (UML) notation showed me how critical formal layout diagrams are to avoiding architectural failures down the line.

### Challenges & Growth
The real growth happened during the group case studies, where we moved away from simple code writing to managing complex project documentation across sequential development phases. Progressing step-by-step through detailed structural submissions starting with project formulation, moving into Software Requirements Specifications (SRS), designing Software Design Descriptions (SDD), and mapping out Software Test Documentation (STD) demanded absolute precision. Balancing technical tools like Figma for design layouts alongside detailed UML structural mapping while handling tight team roles was demanding, but it built up my baseline capacity to manage scale.

### Future Application
The engineering principles embedded in this Work Based Learning curriculum will serve as my blueprint for future software deployment. I will no longer approach system creation as a simple coding exercise, but rather as a highly structured lifecycle. Moving forward into my engineering career, I plan to run rigorous requirement processes, apply solid structural design patterns such as Model-View-Controller (MVC) or layered architectures, and write thorough black-box and white-box test specifications to ensure the delivery of high-performing, production-grade applications.
